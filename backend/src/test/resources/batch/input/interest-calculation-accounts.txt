# Interest Calculation Test Data - Account Records
# This file contains carefully crafted account records for testing the CBACT04C COBOL-to-Java
# interest calculation migration with BigDecimal precision matching COBOL COMP-3 behavior.
#
# Record Format: Fixed-width ASCII matching CVACT01Y copybook layout (300 bytes per record)
# Key fields for interest calculation testing:
# - ACCT-ID: Account identifier (11 digits)
# - ACCT-ACTIVE-STATUS: Account status (Y/N)
# - ACCT-CURR-BAL: Current balance (S9(10)V99 - packed decimal in COBOL)
# - ACCT-CREDIT-LIMIT: Credit limit (S9(10)V99 - packed decimal in COBOL)
# - ACCT-CASH-CREDIT-LIMIT: Cash advance limit (S9(10)V99 - packed decimal in COBOL)
# - ACCT-OPEN-DATE: Account open date (YYYY-MM-DD format)
# - ACCT-EXPIRAION-DATE: Account expiration date (YYYY-MM-DD format)
# - ACCT-REISSUE-DATE: Card reissue date (YYYY-MM-DD format)
# - ACCT-CURR-CYC-CREDIT: Current cycle credit (S9(10)V99 - packed decimal in COBOL)
# - ACCT-CURR-CYC-DEBIT: Current cycle debit (S9(10)V99 - packed decimal in COBOL)
# - ACCT-GROUP-ID: Disclosure group ID (maps to different interest rates)
#
# Test Scenarios Covered:
# 1. Zero balance accounts (no interest calculation)
# 2. Small balance accounts (penny-level precision testing)
# 3. Medium balance accounts (standard calculation testing)
# 4. High balance accounts (maximum precision testing)
# 5. Maximum credit limit accounts (edge case testing)
# 6. Different disclosure groups (A, B, C, DEFAULT) for interest rate variations
# 7. Various cycle dates for interest accrual period testing
# 8. Negative balance accounts (credit balance scenarios)
